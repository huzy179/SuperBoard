## Design Context

### Users

SuperBoard is built for **software developers, product managers, and agile teams** who need a high-performance, Jira-style task management environment. The context is high-frequency usage where speed, clarity, and real-time collaboration are critical.

### Brand Personality

SuperBoard's personality is defined by:

1. **Precise**: Every pixel serves a purpose; no decorative clutter.
2. **Technical**: It feels like a powerful tool for power users, embracing data density without feeling overwhelming.
3. **Fluid**: Real-time updates and optimistic UI make the interface feel alive and responsive.

### Aesthetic Direction

The visual tone is **Refined Technical Minimalism**. It avoids the generic "rounded card" look in favor of sharp typography, high-contrast states, and subtle motion.

- **Theme**: Primarily **Adaptive** (respecting system preferences), but with a focus on a high-contrast dark mode for deep work.
- **References**: Linear, Vercel, and modern dev tools.
- **Anti-references**: Cluttered legacy Jira, generic "AI-generated" purple/blue gradient dashboards.

### Design Principles

1. **Information Density with Breathing Room**: Use a precise 4pt spacing scale to keep things dense but never crowded.
2. **Typography as Hierarchy**: Rely on font weight, size, and careful spacing instead of borders and shadows to define structure.
3. **Intentional Motion**: Use Framer Motion for meaningful state transitions (e.g., drawer openings, task movements) to reinforce the "fluid" personality.
4. **Perceptual Color Consistency**: Use OKLCH for all color tokens to maintain visual harmony across light and dark themes.
