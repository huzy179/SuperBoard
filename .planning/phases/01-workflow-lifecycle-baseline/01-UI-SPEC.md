## Interaction Contract

- Status transition UX is optimistic-first for board drag/drop and list select changes; rollback on API failure and restore previous local state.
- Conflict/stale response handling must show a warning banner, refresh authoritative data immediately, and keep a visible retry action.
- Board and list views must reflect the same status model and stay consistent after mutation and after refresh/focus return.
- Archive model uses soft-delete semantics with default hidden archived records and a visible `Show archived` toggle at list/board scope.
- Restore action is blocked when parent is archived; UI must display ordered guidance: restore workspace/project first, then child item.
- Provide manual refresh control in workflow screens; automatic lightweight refresh triggers on app/window focus.

Source: locked decisions in `01-CONTEXT.md` + architecture recommendations in `01-RESEARCH.md`.
